## 字段说明

1. `Case ID`: 唯一标识，建议 `<DOMAIN>-<number>`
2. `Layer`: `smoke` / `core` / `release`
3. `Platform`: 固定 `ios`
4. `Last Result`: `PASS` / `FAIL` / `SKIP`

## 维护规则

1. 新增功能至少新增或更新 1 条 iOS E2E 并登记。
2. 线上缺陷修复必须补“缺陷复现用例”并登记链接。
3. 连续 3 次 `FAIL` 的用例需在 `Notes` 标注根因和处理计划。
